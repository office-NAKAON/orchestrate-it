# 出力パターン

スキルが一貫した高品質の出力を生成する必要がある場合に使用するパターン。

## テンプレートパターン

出力形式を指定するためのテンプレートを提供する。厳密さのレベルはニーズに合わせて調整。

### 厳密な要件の場合（APIレスポンスやデータ形式など）

```markdown
## レポート構造

必ずこのテンプレート構造を使用：

# [分析タイトル]

## エグゼクティブサマリー
[主要な発見の1段落概要]

## 主要な発見
- 発見1（サポートデータ付き）
- 発見2（サポートデータ付き）
- 発見3（サポートデータ付き）

## 推奨事項
1. 具体的で実行可能な推奨事項
2. 具体的で実行可能な推奨事項
```

### 柔軟なガイダンスの場合（適応が有用な場合）

```markdown
## レポート構造

デフォルト形式だが、状況に応じて判断：

# [分析タイトル]

## エグゼクティブサマリー
[概要]

## 主要な発見
[発見に基づいてセクションを調整]

## 推奨事項
[具体的なコンテキストに合わせて調整]

分析タイプに応じてセクションを調整すること。
```

## 例示パターン

出力品質が例を見ることに依存する場合、入力/出力のペアを提供：

```markdown
## コミットメッセージ形式

以下の例に従ってコミットメッセージを生成：

**例1:**
入力: JWTトークンによるユーザー認証を追加
出力:
```
feat(auth): JWT認証を実装

ログインエンドポイントとトークン検証ミドルウェアを追加
```

**例2:**
入力: レポートで日付が正しく表示されないバグを修正
出力:
```
fix(reports): タイムゾーン変換の日付フォーマットを修正

レポート生成全体でUTCタイムスタンプを一貫して使用
```

このスタイルに従う: type(scope): 簡潔な説明、その後に詳細な説明。
```

例は、説明だけよりも望ましいスタイルと詳細レベルをClaudeが理解するのに役立つ。

## 教育向け出力パターン

### 指導案形式

```markdown
## 指導案テンプレート

**単元名**: [単元名]
**対象学年**: [学年]
**授業時間**: [分]

### 本時の目標
- [目標1]
- [目標2]

### 準備物
- [ ] [準備物1]
- [ ] [準備物2]

### 授業展開

| 時間 | 学習活動 | 指導上の留意点 | 評価 |
|------|---------|--------------|------|
| 導入（5分） | | | |
| 展開（35分） | | | |
| まとめ（5分） | | | |

### 板書計画
[板書の構成図]

### 評価規準
- 知識・技能:
- 思考・判断・表現:
- 主体的に学習に取り組む態度:
```

### 通知表コメント形式

```markdown
**入力例:** 算数が苦手だが、最近九九を頑張っている3年生
**出力例:** 九九の学習に粘り強く取り組み、着実に定着が見られます。計算の速さも向上してきました。

**入力例:** 発表は苦手だが、グループ活動では積極的な5年生
**出力例:** グループ活動では自分の考えを進んで伝え、友達の意見も大切にしながら協力して課題に取り組むことができました。
```
